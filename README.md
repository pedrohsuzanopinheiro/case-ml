# Titanic Survival API

Serverless REST API on AWS that accepts Titanic passenger features, runs a pre-trained Random Forest model, and stores predictions in DynamoDB.

## Architecture

- **API Gateway v1** — imports OpenAPI 3.0 spec with `x-amazon-apigateway-integration`
- **Lambda (Python 3.9)** — single function with route dispatch; model loaded once at module level (warm-start cache)
- **DynamoDB** — on-demand billing (`PAY_PER_REQUEST`), UUID partition key

## Local Testing (no AWS account required)

Tests run entirely inside Docker using **moto** to mock DynamoDB. No AWS credentials needed.

### Step 1 — Download the model

```bash
bash scripts/download_model.sh
```

Expected output:
```

Saved to ./lambda/model.pkl
```

### Step 2 — Run the tests

```bash
bash scripts/test_local.sh
```

This command:
1. Pulls `python:3.11-slim` (first run only, ~50 MB)
2. Installs `scikit-learn==1.1.3`, `pandas==1.5.3`, `moto`, `boto3` inside the container
3. Runs `tests/test_local.py` against a mocked DynamoDB

#### Troubleshooting local tests

**"Cannot connect to the Docker daemon"**
Docker Desktop is not running. Start it and wait for the whale icon in the system tray to stop animating.

**WSL: `execvpe(/bin/bash) failed: No such file or directory`**
Docker Desktop's WSL integration is not enabled. In Docker Desktop → Settings → Resources → WSL Integration, enable integration for your distro and restart Docker Desktop.
Alternatively, run the tests from **Git Bash** instead of WSL.

**`model.pkl not found`**
Run `bash scripts/download_model.sh` first.

**`scikit-learn` version mismatch warning**
The model was trained with `scikit-learn==1.1.3`. If you see a `InconsistentVersionWarning`, the test environment has a different version installed — the Docker container pins the correct version, so tests inside Docker are authoritative.

## Deploy to AWS

### Prerequisites

- AWS credentials configured (`aws configure` or environment variables `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` / `AWS_DEFAULT_REGION`)
- Docker Desktop running (needed for the Lambda ZIP build step)
- Terraform installed

### Step 1 — Download the model (if not done already)

```bash
bash scripts/download_model.sh
```

### Step 2 — Build the Lambda ZIP

```bash
bash scripts/build_package.sh
```

This runs inside Docker on the `public.ecr.aws/lambda/python:3.9` Amazon Linux 2 image to ensure binary compatibility. Output: `build/lambda_package.zip`.

### Step 3 — Deploy

```bash
cd terraform
terraform init
terraform apply
```

Terraform will show you a plan with ~8 resources. Type `yes` to confirm.

After apply, the output will show:
```
api_url = "https://<id>.execute-api.us-east-1.amazonaws.com/v1"
table_name = "titanic-prod-sobreviventes"
```

### Step 4 — Smoke test

```bash
export API_URL=$(terraform output -raw api_url)

# Create a prediction
curl -s -X POST "$API_URL/sobreviventes" \
  -H "Content-Type: application/json" \
  -d '{"Age":29,"Parch":0,"SibSp":1,"Fare":26.5,"Pclass":1,"Sex":"female","Embarked":"S"}' \
  | python -m json.tool

# List predictions
curl -s "$API_URL/sobreviventes" | python -m json.tool

# Get by ID (replace <id> with the id returned above)
curl -s "$API_URL/sobreviventes/<id>" | python -m json.tool

# Delete
curl -s -o /dev/null -w "%{http_code}" -X DELETE "$API_URL/sobreviventes/<id>"
# → 204
```

### Tear down

```bash
cd terraform
terraform destroy
```

## API Contract

| Method | Path | Body | Success | Error |
|--------|------|------|---------|-------|
| `POST` | `/sobreviventes` | JSON passenger (see below) | `201` `{id, survival_probability}` | `400` missing fields |
| `GET` | `/sobreviventes` | — | `200` array of items | — |
| `GET` | `/sobreviventes/{id}` | — | `200` item | `404` |
| `DELETE` | `/sobreviventes/{id}` | — | `204` | `404` |

### POST request body

```json
{
  "Age":     29,
  "Parch":   0,
  "SibSp":   1,
  "Fare":    26.5,
  "Pclass":  1,
  "Sex":     "female",
  "Embarked":"S"
}
```

| Field | Type | Values |
|-------|------|--------|
| `Age` | number | passenger age |
| `Parch` | integer | number of parents/children aboard |
| `SibSp` | integer | number of siblings/spouses aboard |
| `Fare` | number | ticket fare |
| `Pclass` | integer | `1`, `2`, or `3` |
| `Sex` | string | `"male"` or `"female"` |
| `Embarked` | string | `"C"` (Cherbourg), `"Q"` (Queenstown), `"S"` (Southampton) |

## Configuration

Default values (override in `terraform/variables.tf` or via `-var` flag):

| Variable | Default | Description |
|----------|---------|-------------|
| `region` | `us-east-1` | AWS region |
| `project_name` | `titanic` | Resource name prefix |
| `environment` | `prod` | Environment suffix |

Example override:
```bash
terraform apply -var="region=sa-east-1" -var="environment=dev"
```
